# User Flows - Регата Трекер

## Навигационная структура

```
/ (Root)
├── /login
├── /register
├── /design-system
│
├── /judge (Судья)
│   ├── Dashboard
│   ├── /race/new (Создать гонку)
│   ├── /race/:id (Детали гонки)
│   ├── /course/new (Создать трассу)
│   └── /course/:id (Редактировать трассу)
│
└── /participant (Участник)
    ├── Dashboard
    ├── /race/:id (Просмотр гонки)
    └── /racing/:id (Активная гонка - hands-free)
```

## Flow 1: Судья - Создание и проведение гонки

### Сценарий: Судья создает новую гонку и проводит её

```
1. Старт
   ↓
2. / (Role Select)
   → Выбрать "Судья"
   ↓
3. /judge (Dashboard)
   → Нажать "Новая гонка"
   ↓
4. /judge/race/new (Create Race)
   → Заполнить:
      - Название гонки
      - Выбрать существующую трассу ИЛИ "Создать новую"
      
   [Если создать новую трассу]
   ↓
5. /judge/course/new (Course Builder)
   → Ввести название трассы
   → Выбрать тип гонки (W/L, Match Race)
   → Нажимать на карту для добавления точек:
      - Start line (2 точки)
      - Marks
      - Gates (опционально)
      - Finish line (2 точки)
   → Валидация автоматическая (показывает ошибки)
   → "Сохранить"
   ↓
   [Возврат к Create Race]
   
4b. /judge/race/new (продолжение)
   → Выбрать созданную трассу
   → Установить время старта
   → Выбрать GPS интервал (3/5/10/15 сек)
   → "Создать гонку"
   ↓
6. /judge (Dashboard)
   → Гонка появилась в списке "Предстоящие"
   → Статус: "Запланирована"
   → Нажать на карточку гонки
   ↓
7. /judge/race/:id (Race Details)
   → Информация о гонке
   → Список участников
   → Статус: "Готова к старту" (когда подтвердили участники)
   → Нажать "НАЧАТЬ ГОНКУ" (зеленая кнопка XL)
   ↓
8. Гонка началась
   → Статус: "В процессе"
   → Участники получают уведомление
   → Начинается автоматический GPS трекинг
   → Показывается список участников с позициями
   ↓
9. Ожидание финиша участников
   → Участники финишируют автоматически (по GPS)
   → Обновляется список результатов
   ↓
10. Все финишировали
    → Нажать "ЗАКОНЧИТЬ ГОНКУ" (красная кнопка XL)
    ↓
11. /judge/race/:id (результаты)
    → Статус: "Завершена"
    → Таблица результатов с позициями и временем
    → Возможность вернуться на Dashboard
```

## Flow 2: Судья - Быстрое создание трассы

### Сценарий: Судья хочет создать трассу для будущих гонок

```
1. /judge (Dashboard)
   → Нажать "Трасса"
   ↓
2. /judge/course/new
   → Ввести название: "Олимпийская W/L"
   → Выбрать тип: "Windward-Leeward"
   → Добавить точки:
      1-2. Start line (зеленые)
      3. Windward mark (синяя)
      4. Offset mark (фиолетовая)
      5-6. Leeward gate (оранжевые)
      7-8. Finish line (красные)
   → Валидация: ✓ Трасса валидна
   → "Сохранить"
   ↓
3. /judge (Dashboard)
   → Трасса сохранена
   → Доступна для выбора при создании гонки
```

## Flow 3: Участник - Подготовка к гонке

### Сценарий: Участник получил приглашение на гонку

```
1. Старт
   ↓
2. / (Role Select)
   → Выбрать "Участник"
   ↓
3. /participant (Dashboard)
   → Видит гонку в разделе "Предстоящие"
   → Статус: "Запланирована"
   → Нажать на карточку
   ↓
4. /participant/race/:id
   → Информация о гонке:
      - Название
      - Трасса
      - Время старта
      - GPS интервал
   → Моя информация:
      - Яхта: "Буря"
      - Парусный номер: RUS-101
      - Статус: "Приглашен"
   → Кнопки: "ПРИНЯТЬ" / "ОТКЛОНИТЬ"
   → Нажать "ПРИНЯТЬ"
   ↓
5. Участие подтверждено
   → Статус изменился: "Подтвержден"
   → Показаны инструкции:
      • Убедитесь, что GPS включен
      • Зарядите телефон
      • При старте получите уведомление
      • Во время гонки не трогайте телефон
   ↓
6. Возврат на Dashboard
   → Гонка в разделе "Предстоящие"
   → Бейдж: "Подтвержден"
```

## Flow 4: Участник - Участие в гонке

### Сценарий: Участник участвует в активной гонке

```
1. Подготовка
   → Участник на воде с заряженным телефоном
   → GPS включен
   → Приложение открыто (или в фоне)
   ↓
2. Судья начинает гонку
   → Участник получает PUSH уведомление
   → "Гонка 'Весенний Кубок' началась!"
   ↓
3. /participant (Dashboard)
   → Гонка перемещена в "Активные гонки"
   → Яркий зеленый блок:
      "АКТИВНАЯ ГОНКА"
      [Перейти к гонке]
   → Нажать "Перейти к гонке"
   ↓
4. /participant/racing/:id (HANDS-FREE MODE)
   
   Экран показывает:
   ┌─────────────────────────┐
   │ Весенний Кубок          │
   │ Режим гонки             │
   ├─────────────────────────┤
   │                         │
   │   Текущая позиция       │
   │         2               │ ← ОЧЕНЬ КРУПНО (7xl)
   │       из 4              │
   │                         │
   │   Прошло времени        │
   │      15м 32с            │ ← КРУПНО (5xl)
   │                         │
   │ ●━━━━━━━━━━━━━━━━━━━   │ ← Зеленая точка (пульсирует)
   │ GPS трекинг             │
   │ Каждые 5с               │
   │                         │
   │ ┌─────────┬─────────┐   │
   │ │ 8.5 kn  │  45°    │   │
   │ │ Скорость│  Курс   │   │
   │ └─────────┴─────────┘   │
   │                         │
   └─────────────────────────┘
   
   → НЕТ ВЗАИМОДЕЙСТВИЯ
   → Телефон автоматически:
      - Отправляет GPS каждые 5 сек
      - Обновляет позицию
      - Засекает время
      - Показывает скорость/курс
   ↓
5. Пересечение финишной линии
   → Определяется автоматически по GPS
   → Отправка события финиша на сервер
   → Статус: "Финишировал"
   ↓
6. Автоматический переход
   → Экран обновляется:
      "ВЫ ФИНИШИРОВАЛИ!"
      Позиция: 2
      Время: 15:32
   → Можно вернуться на Dashboard
   ↓
7. /participant (Dashboard)
   → Гонка перемещена в "Завершенные"
   → Показана позиция: "2 место"
```

## Flow 5: Edge Cases и состояния

### 5.1: Потеря GPS во время гонки

```
/participant/racing/:id
→ GPS становится недоступен
→ Индикатор меняется:
   ⚠━━━━━━━━━━━━━━━━━━━
   GPS трекинг
   Не активен
→ Попытка переподключения каждые 5 сек
→ При восстановлении:
   ●━━━━━━━━━━━━━━━━━━━
   GPS трекинг
   Восстановлен
```

### 5.2: Потеря интернета во время гонки

```
/participant/racing/:id
→ Нет интернета
→ GPS точки сохраняются локально в очередь
→ Показывается индикатор:
   "Очередь: 15 точек"
→ При восстановлении:
   "Отправка очереди..."
   → Все точки отправляются
   → "Очередь отправлена ✓"
```

### 5.3: Судья завершает гонку до финиша всех участников

```
/judge/race/:id
→ Судья нажимает "ЗАКОНЧИТЬ ГОНКУ"
→ Подтверждение:
   "Не все участники финишировали.
    Завершить гонку?"
   [Отмена] [Завершить]
→ При завершении:
   - Незавершенные получают статус DNF
   - Результаты фиксируются
   - Уведомления участникам
```

### 5.4: Создание трассы с ошибками валидации

```
/judge/course/new
→ Добавлено всего 1 точка (Start)
→ Валидация показывает:
   ⚠ Ошибки:
   • Минимум 2 точки необходимо
   • Требуется финишная линия
→ Кнопка "Сохранить" disabled
→ Добавление Finish point
→ Валидация:
   ✓ Трасса валидна
→ "Сохранить" становится активной
```

### 5.5: Участник отклоняет приглашение

```
/participant/race/:id
→ Нажать "ОТКЛОНИТЬ"
→ Подтверждение:
   "Вы уверены, что не хотите
    участвовать в гонке?"
   [Назад] [Отклонить]
→ Статус: "Отклонено"
→ Гонка удаляется из списка участника
→ Судья видит обновленный список участников
```

## Навигационные паттерны

### Back Navigation
- **Все экраны**: кнопка "← Назад" в header
- Возврат на предыдущий экран
- Сохранение состояния где нужно

### Bottom Navigation
- **НЕ используется** (single-task focus)
- Вместо этого: Dashboard как hub

### Modal Dialogs
- Подтверждения опасных действий
- Формы создания (fullscreen)

### Pull to Refresh
- На Dashboard экранах
- Обновление списка гонок

### Deep Links
- `/race/:id` - прямая ссылка на гонку
- `/racing/:id` - прямой переход в активную гонку
- Из push-уведомлений

## Состояния загрузки

### При входе в приложение
```
/ → Loading spinner 500ms → Role Select
```

### При загрузке Dashboard
```
/judge → LoadingView → Data loaded → Dashboard content
```

### При создании гонки
```
Create Race form → Submit → Button loading state → Success → Navigate to Dashboard
```

### При старте гонки
```
Race Details → "Начать гонку" → Loading (1-2s) → Status updated → Active state
```

## Transitions

### Screen Transitions
- Slide from right (forward navigation)
- Slide from left (back navigation)
- Fade (modal dialogs)

### Component Transitions
- Buttons: scale 0.95 on press
- Cards: subtle shadow increase on hover
- Badges: fade in
- Loading spinners: rotate animation

## Notifications Flow

### Push Notification → App

```
1. Гонка начинается
   → Server sends push
   ↓
2. Participant receives:
   "Гонка 'Весенний Кубок' началась!"
   [Открыть]
   ↓
3. Tap notification
   → Deep link to /participant/racing/:id
   → Opens hands-free mode
```

## Summary: Critical User Journeys

1. **Судья создает и проводит гонку**: ~ 5 минут
2. **Участник подтверждает участие**: ~ 30 секунд
3. **Участник в режиме гонки**: 0 действий (автомат)
4. **Просмотр результатов после финиша**: ~ 10 секунд
5. **Создание сложной трассы**: ~ 3-5 минут
