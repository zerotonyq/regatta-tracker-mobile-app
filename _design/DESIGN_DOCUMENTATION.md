# Регата Трекер - Документация дизайна

## Обзор проекта

Мобильное приложение для трекинга парусных регат с двумя ролями пользователей:
- **Судья** - создание гонок, управление трассами, контроль соревнований
- **Участник** - участие в гонках с автоматическим GPS трекингом

## Дизайн-философия

### Принципы
- **Функциональность превыше всего** - минимум декора, максимум пользы
- **Sport-tech стиль** - современный, технологичный, профессиональный
- **Читаемость на солнце** - высокий контраст, крупные элементы
- **Hands-free во время гонки** - нет взаимодействия с телефоном на воде
- **Масштабируемость** - компонентная система, готовая к расширению

### Условия использования
- Яркое солнце
- Мокрые руки
- Плохой интернет
- Движение на воде
- Минимальная когнитивная нагрузка

## Компонентная система

### Button
**Варианты**:
- `primary` - основные действия (синий фон)
- `secondary` - вторичные действия (серый фон)
- `success` - подтверждающие действия (зеленый)
- `danger` - опасные действия (красный)
- `ghost` - прозрачные кнопки
- `outline` - с обводкой

**Размеры**:
- `sm` - 40px высота
- `md` - 48px высота (стандарт)
- `lg` - 56px высота
- `xl` - 64px высота (для критических действий)

**Состояния**:
- Normal
- Pressed (scale 0.95)
- Disabled (opacity 0.5)
- Loading (спиннер)

### Badge
**Использование**: статусы, метки, счетчики

**Варианты**:
- `default`, `success`, `warning`, `danger`, `info`
- `planned`, `active`, `finished` (для статусов гонки)

### Card
**Использование**: контейнеры для контента

**Свойства**:
- Белый фон
- Тень: shadow-sm
- Border radius: 12px
- Padding: 16px
- `interactive` - для кликабельных карточек (hover, active эффекты)

### Input
**Свойства**:
- Минимальная высота: 48px
- Border radius: 12px
- Focus state: ring эффект
- Error state: красная обводка
- Disabled state: opacity 0.5

## Экраны и User Flows

### Общая структура экрана
```
┌─────────────────────────┐
│   Header (Primary)      │ ← Цветной header с навигацией
├─────────────────────────┤
│                         │
│   Content Area          │ ← Белый/серый фон, max-width 672px
│   (max-w-2xl)          │
│                         │
└─────────────────────────┘
```

### Судья - User Flow

1. **Onboarding / Выбор роли**
   - Выбор "Судья"
   - → Dashboard Судьи

2. **Dashboard Судьи**
   - Быстрые действия: "Новая гонка", "Трасса"
   - Список активных гонок
   - Список предстоящих гонок
   - Список завершенных гонок

3. **Создание гонки**
   - Название гонки
   - Выбор трассы (или создание новой)
   - Запланированное время старта
   - GPS интервал (3/5/10/15 сек)
   - → Сохранение → Dashboard

4. **Конструктор трассы**
   - OpenStreetMap интеграция
   - Типы точек:
     - Start line (зеленая)
     - Finish line (красная)
     - Mark (синяя)
     - Gate left/right (оранжевые)
     - Offset mark (фиолетовая)
   - Валидация трассы (правила World Sailing)
   - Сохранение

5. **Детали гонки**
   - Информация о гонке
   - Список участников
   - Кнопка "Начать гонку" (зеленая, XL)
   - Кнопка "Закончить гонку" (красная, XL)
   - Результаты (после финиша)

### Участник - User Flow

1. **Onboarding / Выбор роли**
   - Выбор "Участник"
   - → Dashboard Участника

2. **Dashboard Участника**
   - Активная гонка (если есть) - яркий блок
   - Предстоящие гонки
   - Завершенные гонки

3. **Просмотр гонки (до старта)**
   - Детали гонки
   - Подтверждение участия (Принять/Отклонить)
   - Информация о трассе
   - Другие участники
   - Инструкции по подготовке

4. **Режим гонки (Hands-free)**
   - КРУПНАЯ текущая позиция (7xl size)
   - КРУПНОЕ прошедшее время (5xl size)
   - GPS статус (анимированная точка)
   - Скорость / Курс
   - Автоматическая отправка GPS
   - Без взаимодействия пользователя

5. **После гонки**
   - Результаты
   - Позиция
   - Время финиша

### Состояния экранов

#### Loading State
- Спиннер по центру
- Текст "Загрузка..."
- Минимальная высота 400px

#### No Internet
- Иконка WifiOff (оранжевая)

#### Error State
- Иконка AlertCircle (красная)
- Заголовок ошибки
- Описание
- Кнопка "Повторить"

#### Empty State
- Кастомная иконка
- Заголовок
- Описание

#### Background Tracking Active
- Зеленая анимированная точка
- Текст "Фоновый трекинг активен"
- Compact view

### Валидация
Проверки по правилам World Sailing:
- ✓ Минимум 2 точки
- ✓ Наличие стартовой линии

- ✓ Для W/L: верхняя метка обязательна
- ✓ Ворота требуют обе метки (left + right)

## API Integration Points

### Backend Endpoints (ориентир)
```
/auth
  - POST /login
  - POST /register
  - POST /refresh

/receiver
  - POST /upload-location

/management
  - POST /create-race
  - POST /start-race
  - POST /end-race
  - GET /races
  - GET /race/:id
```

### Обработка состояний
- Success: 200-299
- Client errors: 400-499
- Server errors: 500-599
- Network errors: показать NoInternetView
- Retry logic с exponential backoff

### Offline Queue
- GPS точки сохраняются локально
- Отправка при восстановлении связи
- Индикатор "очередь отправки"

## Открытые вопросы / Assumptions

### Реализовано как assumptions:
1. **GPS интервал**: пользователь может выбрать 3/5/10/15 сек
2. **Автофиниш**: определяется по пересечению GPS-трека с финишной линией
3. **Уведомления**: участники получают push при старте гонки
4. **Фоновый трекинг**: работает даже при блокировке экрана (требует native permissions)
5. **Offline queue**: GPS точки сохраняются и отправляются при появлении сети

### Требуют уточнения от продукта:
1. Как обрабатывать DNF (Did Not Finish)?
2. Нужна ли функция "снять с гонки" для судьи?
3. Как работать с multiple laps (несколько кругов)?
4. Приглашение участников - email/SMS/in-app?
5. Права доступа к closed races?
6. Экспорт результатов (PDF/CSV)?
7. Live tracking на карте во время гонки?
8. Replay треков после финиша?
